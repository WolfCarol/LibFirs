FirsLib是一个极简的2D游戏开发库，包装了一些win32api，功能有限，只完成了一些不得不和系统打交道的活，编译成动态链接库只有25kb不到，我用来给c#做p/invoke绑定的，因为我实在不愿意继续用c/cpp开发游戏。我不确定是否有bug，只简单测了一些，我会在后续实际项目开发中使用这个库，并且修复漏洞和更新一些必要的新功能。

我有计划在这个库的基础上用c#写一个2D游戏开发框架，功能会相对完善，但我不能保证时限。



这个库的主要功能有以下几个方面：

1创建和管理窗口（win32api）

2获取键盘/鼠标/手柄输入（xinput）

3用opengl绘制图像

4用xaudio2播放音频



我认为这些是开发一个常见的2D游戏必不可少的最基础的功能，而这些又不得不和底层系统打交道，这也是常见的多媒体库正在做的事。

我厌烦臃肿庞大的游戏引擎，因为我所要开发的游戏非常简陋以至于根本用不上那些功能，我从游戏引擎转到游戏开发框架转到多媒体开发库，最后选择了自己包装系统底层的实现细节，做一个自己用得上的精简库，所有代码包含在一个头文件内，总共不超过1500行，但这个耗费了我相当长的时间，因为我既不熟悉c/cpp也不熟悉win32api，这对我来说是个相当大的挑战。



如果有人想要使用或者从这个库中学习，我非常欢迎，我声明了MIT License和Public Domain两种许可方式，我无心处理许可授权和法律纠纷。

我并不熟悉linux也没用过macos，因此我无力支持除了windows以外的平台。



这个库不会比现有的成熟的游戏/多媒体开发库更好，但是我偏爱它的小巧。



我以我的挚友的名字为这个库命名，在此我要向她表示感谢。
